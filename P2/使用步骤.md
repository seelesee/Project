# STM32 智能温湿度监测系统 — 使用步骤

> 工具链: **STM32CubeMX** → **Keil uVision5** → **Proteus 8 Professional**

---

## 第一步：STM32CubeMX 配置工程

### 1.1 打开工程
```
双击 STM32_TempHumidity.ioc
```

### 1.2 检查引脚配置 (Pinout 视图)

| 引脚 | 功能 | 模式 |
|------|------|------|
| **PA0** | DHT22 DATA | GPIO_Output — 开漏输出 |
| **PA1** | KEY UP | GPIO_Input — 下拉 (Pull-down) |
| **PA2** | KEY DOWN | GPIO_Input — 下拉 |
| **PA3** | KEY SET | GPIO_Input — 下拉 |
| **PA4** | BUZZER | GPIO_Output — 推挽输出 |
| **PA9** | USART1_TX | Alternate Function Push-Pull |
| **PA10** | USART1_RX | Input — 浮空 |
| **PB6** | OLED SCL | GPIO_Output — 推挽输出 |
| **PB7** | OLED SDA | GPIO_Output — 推挽输出 |
| **PA13** | SWDIO | Serial_Wire (调试口，保留) |
| **PA14** | SWCLK | Serial_Wire (调试口，保留) |

### 1.3 检查时钟 (Clock Configuration)

```
HSE: 8 MHz
  → PLL Source Mux → PLLMUL ×9
  → System Clock Mux
SYSCLK: 72 MHz
  ├── AHB ÷1  → HCLK  72 MHz
  ├── APB1 ÷2 → 36 MHz
  └── APB2 ÷1 → 72 MHz
Flash Latency: 2 wait states
```

### 1.4 检查 USART1 (Connectivity 视图)

```
Mode: Asynchronous
Baud Rate: 115200  /  Word Length: 8  /  Parity: None  /  Stop Bits: 1
```

### 1.5 检查 NVIC 中断

```
☑ USART1 global interrupt  → Enabled
☑ SysTick timer            → Enabled
```

### 1.6 工程设置 (Project Manager)

```
Project Name:  STM32_TempHumidity
Toolchain/IDE: MDK-ARM V5
```

### 1.7 生成代码

```
点击右上角 [GENERATE CODE] → 等待完成 → 关闭 CubeMX
```

> ⚠️ 生成后会覆盖 `Core/Src/main.c`，用本项目 `main.c` 替换回去。

## 第二步：Keil uVision5 编译

### 2.1 打开工程

```
双击 MDK-ARM\STM32_TempHumidity.uvprojx
```

### 2.2 配置头文件路径

```
菜单 → Project → Options for Target → C/C++(AC6) → Include Paths:
```
```
..
../Inc
../Drivers/CMSIS/Include
../Drivers/CMSIS/Device/ST/STM32F1xx/Include
../Drivers/STM32F1xx_HAL_Driver/Inc
../Core/Inc
```

### 2.3 配置预定义宏

```
同一个 C/C++ 标签 → Define 框输入:
USE_HAL_DRIVER,STM32F103xB
```

### 2.4 勾选生成 HEX

```
同一个窗口 → Output 标签:
☑ Create HEX File
```

### 2.5 编译

```
按 F7 → 等待 Build Output 窗口显示:
"0 Error(s), 0 Warning(s)"
```

### 2.6 确认输出

```
HEX 文件位置: MDK-ARM\STM32_TempHumidity\STM32_TempHumidity.hex
```

## 第三步：Proteus 8 仿真

### 3.1 打开仿真工程

```
Proteus 8 → File → Open Project → 选 Proteus\STM32_TempHumidity.pdsprj
```

### 3.2 加载固件

```
双击原理图中的 STM32F103C8 芯片 (U1)
→ Program File: 浏览选择第二步生成的 .hex
→ Clock Frequency: 72000000
→ OK
```

### 3.3 配置虚拟终端

```
双击 Virtual Terminal → 设置:
Baud: 115200 / Data: 8 / Parity: NONE / Stop: 1
```

### 3.4 启动仿真

```
点击左下角 ▶ (Run)
```

### 3.5 观察现象

| 阶段 | 现象 |
|------|------|
| 0~2秒 | OLED 显示开机 Logo "温湿度监测系统" |
| 2秒后 | 主界面实时显示温湿度 "Temp: xx.x°C / Humi: xx.x%" |
| 按 **SET** | 进入阀值设定，当前编辑项闪烁 |
| 按 **UP/DOWN** | 数值 ±0.5 |
| 再按 **SET** | 切换到下一个阀值项 (温度上限→下限→湿度上限→下限→退出) |
| 超限报警 | OLED 显示 "报警" + 蜂鸣器间歇响 + 串口上报报警帧 |

## 按键操作速查

| 操作 | 正常模式 | 设定模式 |
|------|---------|---------|
| 单击 **SET** | 进入设定模式 | 切换下一个设定项 |
| 单击 **UP** | 手动上传一次数据 | 当前值 +0.5 |
| 单击 **DOWN** | 静音报警 | 当前值 -0.5 |
| 长按 **SET** | 恢复默认阀值 | 保存并退出 |

## 常见问题

| 问题 | 解决 |
|------|------|
| Keil 找不到 `stm32f1xx_hal.h` | 检查 **2.2** 头文件路径是否完整 |
| Keil 编译报 `#error` 宏未定义 | 检查 **2.3** 预定义宏 `USE_HAL_DRIVER,STM32F103xB` |
| Proteus OLED 不显示 | 确认 **3.2** Clock Frequency 设为 `72000000` |
| 虚拟终端乱码 | 确认 **3.3** 波特率 `115200-8-N-1` |
| 找不到 HEX 文件 | 检查 **2.4** `Create HEX File` 是否勾选 |
